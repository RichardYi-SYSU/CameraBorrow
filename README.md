# Camera Borrow Demo

面向校内工作室设备借还场景的最小闭环演示系统。当前版本基于 `Next.js` 实现，重点展示“申请借用 -> 管理员审批 -> 借出确认 -> 用户归还 -> 管理员完成归还”的核心流程。

## 功能概览

- 普通用户视角：查看设备、提交借用申请、查看自己的申请状态、发起归还。
- 管理员视角：审批申请、拒绝申请、确认借出、确认归还。
- 状态流转：`pending -> approved -> borrowed -> returned_pending_confirm -> completed`。
- 数据存储：使用本地 `data/demo-data.json`，适合 demo 演示，不依赖数据库。

## 本地运行

环境要求：

- Node.js `>= 20`
- npm `>= 10`

启动步骤：

```bash
npm install
npm run dev
```

打开 `http://localhost:3000`。

生产检查：

```bash
npm run lint
npm run build
```

## 演示建议流程

1. 以“普通用户视角”选择设备并提交借用申请。
2. 切换到“管理员视角”，对该申请执行批准和借出确认。
3. 切回“普通用户视角”，在申请记录中点击“发起归还”。
4. 再切回“管理员视角”，完成归还确认。
5. 观察设备状态和申请时间线的同步变化。

## 目录结构

```text
app/            页面与 API 路由
components/     前端展示与交互组件
lib/            类型定义、状态流转与本地数据读写
data/           Demo 数据文件
docs/           需求文档
```

## 当前限制

- 不包含真实登录、角色权限、消息通知和统计报表。
- 不接数据库，服务重启后仍依赖本地 JSON 文件。
- 当前为单设备单申请模型，优先保证演示闭环清晰。
